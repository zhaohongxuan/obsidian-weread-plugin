## 1.0.0 - 2026年4月3日

### 🎉 Major Release - 完整的微信读书书架主页

**版本升级:** 0.17.1 → 1.0.0

#### ✨ 新增功能

##### 1. 📚 微信读书书架视图 (WeRead Bookshelf View)
- **完整的书架浏览界面**：展示用户所有书籍的列表和卡片视图
- **书籍元数据显示**：显示书籍封面、标题、作者、划线数、想法数等
- **最近阅读追踪**：显示每本书的最近阅读时间
- **阅读状态标签**：根据本地文件的 `finishedDate` 元数据区分"在读"和"已读"
![image|700](https://raw.githubusercontent.com/zhaohongxuan/picgo/master/assets/images/2026-04-03_10-17-53_yevs7.png)
##### 2. 🔍 书架筛选和排序功能
- **多维度筛选**：
  - 📖 书籍类型筛选（全部/图书/公众号）
  - 🔄 同步状态筛选（全部/仅远程/已同步/仅本地）
  - 📕 **阅读状态筛选（在读+已读/在读/已读）** ✨ 新功能
  - 🔎 实时搜索（书名和作者）
  
- **排序选项**：
  - ⏰ 按最近阅读时间排序（支持按年份分组）
  - 📝 按书名排序

##### 3. 📖 书籍详情模态窗口 (Book Detail Modal)
- **完整的书籍信息展示**：
  - 书籍封面、标题、作者
  - 同步状态徽章
  - 阅读进度和完成状态
  - 划线、想法、阅读时长等统计信息
  - 出版社、出版时间、分类、字数等出版信息
  - 书籍简介

- **交互功能**：
  - 点击本地文件按钮直接打开本地笔记
  - 打开网页版详情的快捷链接（桌面版）
  - 同步此书籍（仅远程书籍）
  - 删除本地文件（仅本地书籍）

##### 4. 🌐 网页版打开方式配置 ✨ 新功能
- **新的设置选项**：网页版打开方式（新标签页/新窗口）
- **应用场景**：
  - 书架中的"进入网页版"按钮
  - 书籍详情页中的"打开网页版详情"链接
  - 支持根据用户偏好灵活选择

- **实现方式**：
  - TAB 模式：在 Obsidian 内的新标签页打开
  - WINDOW 模式：使用 Electron 独立窗口打开（提高交互性）

##### 5. 🛠️ Electron 浏览器窗口集成
- **独立浏览器窗口**：解决 Obsidian popout leaf 中 webview 的交互问题
- **Cookie 会话管理**：自动同步插件 Cookie 到窗口会话
- **持久化分区**：使用 `persist:weread-plugin-browser` 分区保持登录状态

##### 6. 🔐 同步模式升级 (Sync Mode System)
- **黑名单模式** (默认)：排除不需要同步的书籍
- **白名单模式** (新功能)：仅同步指定的书籍
![image|700](https://raw.githubusercontent.com/zhaohongxuan/picgo/master/assets/images/2026-04-03_10-22-30_96rum.png)
- **灵活的书籍选择器**：
  - 搜索和筛选界面
  - 预览已选/已排除的书籍
  - 自动排除的书籍（公众号、划线不足）会单独展示
  - 批量操作支持（选中当前结果、清空选择）

##### 7. 📋 单本书籍同步功能
- **快捷同步**：直接从书架同步单本书籍，无需全量同步
- **用户体验改进**：仅远程书籍显示"同步"按钮
- **性能优化**：异步加载书籍列表，不阻塞 UI

##### 8. 🎯 改进的阅读状态识别
- **本地缓存优先**：使用本地文件 frontmatter 中的 `finishedDate` 判断阅读状态
- **性能优势**：避免频繁调用 `getProgress` 接口
- **实时标签**：书架视图中直观展示"在读"和"已读"状态

#### 🔄 UI/UX 改进

##### 工具栏优化
- **新图标按钮**：
  - 📗 同步按钮：同步图标 + "同步"文字
  - ⚙️ 选项按钮：齿轮图标 + "选项"文字
  - 🌐 网页版按钮：地球图标 + "网页版"文字（桌面版专用）

- **响应式设计**：
  - 搜索框宽度优化（200px）
  - 工具栏自适应换行
  - 移动设备专用 UI（隐藏网页版按钮）

##### 快捷菜单更新
- 移除了"在新标签页打开"和"在窗口打开"的菜单项
- 使用新的书架视图和配置方式替代
- 统一的快捷菜单结构

#### 🏗️ 架构改进

##### 新增模块
- `src/bookshelf.ts` - 书架服务
- `src/components/wereadBookshelf.ts` - 书架视图
- `src/components/wereadBookDetailModal.ts` - 详情模态窗口
- `src/components/wereadBrowserWindow.ts` - 浏览器窗口
- `src/syncFilter.ts` - 同步过滤系统
- `src/utils/bookIdUtils.ts` - 书籍 ID 工具

##### 核心改进
- 新增 `bookshelfService` 实例和 `fileManager` 引用
- 扩展 `WereadPluginSettings` 类型（`syncMode`, `readingOpenMode`, `notesWhitelist`）
- 增强 `AnnotationFile` 数据模型支持阅读状态
- 新增 `BookshelfBook`, `BookshelfProgress`, `SyncFilterResult` 类型

#### 🐛 Bug 修复
- 修复 popout leaf 中 webview 的交互问题
- 改进 Cookie 持久化在网页导航中的处理
- 优化同步过滤逻辑的准确性
- 修复设置面板自动滚动问题

#### 📱 平台特性
- **桌面应用**：完整的书架视图、网页版打开方式设置、WINDOW 模式
- **移动应用**：简化的书架视图、隐藏网页版按钮

## 0.17.0 

### 核心改进

#### 🚀 移动端支持增强
- **完整的移动端 UI 适配**：优化登录/登出界面，移动端用户可以清晰查看用户信息和 Cookie 状态
- **移动端 Cookie 管理**：
  - 支持在电脑端重新登录后同步更新
  - 移动端同步时显示友好的操作提示
- **解决 iOS Cookie 中文字符问题**：对中文字符进行 URL 编码处理，修复 iOS 端 401 错误

#### ✨ 设置界面优化
- **用户头像展示**：登出面板展示用户微信读书头像，提升视觉效果
- **Cookie 状态管理**：
  - 移动端登出面板新增 Cookie 状态显示（有效/失效/未登录）
  - 显示上次 Cookie 刷新时间
  - 扫码登录方式下，统一由登录/登出面板展示状态
  
- **登录面板重构**：
  - 整体卡片式设计，包含标题、说明和按钮
  - 桌面端和移动端分别优化布局
  - 提升用户体验

#### 🔄 注销流程自动化
- **自动化登出**：点击注销按钮自动调用微信读书注销 API，无需手动操作
- **简化用户流程**：移除复杂的手动操作步骤，一键完成登出

#### 🔧 API 和Cookie 管理改进
- **Cookie 刷新机制优化**：
  - `refreshCookie()` 新增 `showNotice` 参数，控制提示显示
  - 自动刷新时不显示通知，手动刷新时显示
  - 支持 CookieCloud 刷新后更新刷新时间

- **Cookie 有效性验证增强**：
  - 初始化时自动验证 Cookie 有效性
  - 详细的错误日志和调试信息
  - 区分桌面端和移动端的不同处理逻辑

- **登录窗口 Cookie 同步**：
  - 自动从登录窗口抓取 Cookie，优化登录体验
  - 多个事件触发点，确保 Cookie 同步完整性

#### 📋 其他改进
- **平台检测增强**：登录设置仅在桌面端显示，移动端隐藏
- **日志优化**：增加详细的调试日志，便于问题排查
- **类型安全**：改进 TypeScript 类型定义和类型检查

#### 🎯 用户建议
- 首次使用建议在桌面端（Windows/macOS/Linux）登录和配置
- 移动端可以登录后使用同步功能，但不建议切换登录方式
- 如遇到 Cookie 失效，建议在桌面端重新登录以刷新 Cookie
